# Skill 自动提取系统 — 产品规格

> 一句话：从团队产品文档中自动分析并提取可复用的 AI Agent Skill
> 创建时间：2026-02-15
> 状态：草稿

## 用户是谁

产品经理 + 开发者团队，想把团队沉淀的业务知识变成可复用的 AI 指令。

## 痛点

- 团队有很多需求文档、会议记录、PRD，但这些知识只能人肉阅读理解
- 每个新成员都需要重新学习团队惯例和流程
- 好的实践和方法论只存在于少数人脑子里，无法复用
- 手动写 Skill 太麻烦，很多人宁可复制粘贴

## 核心功能（最多 3 个）

### 功能 1: 文档解析与 Skill 提取
- 用户操作：上传产品文档（PRD、设计稿、会议纪要）或提供文档目录
- 系统响应：AI 分析文档内容，识别可复用的工作流程和指令模式，生成 Skill 草稿
- 成功标准：从 1 份 PRD 中提取 3-5 个可用 Skill

### 功能 2: Skill 编辑与定稿
- 用户操作：查看 AI 生成的 Skill 草稿，编辑修改
- 系统响应：展示提取的 Skill，支持编辑、删除、调整格式
- 成功标准：用户可以编辑 Skill 内容并保存

### 功能 3: 团队共享与版本管理
- 用户操作：把定稿的 Skill 分享给团队成员
- 系统响应：支持 Skill 的版本历史，团队成员可以查看和使用
- 成功标准：团队成员可以搜索和使用其他人提取的 Skill

## 明确不做

- 不做复杂的工作流编排（只提取单个 Skill）
- 不做与外部 AI 平台的深度集成（只输出 Skill 文件）
- 不做代码级别的 Skill（专注产品/需求类 Skill）
- 不支持实时协作编辑（草稿本地保存）

## 竞品概览

| 产品 | 做了什么 | 没做什么 | 我们的机会 |
|------|---------|---------|-----------|
| Langfuse | Prompt 版本管理、测试 | 从文档提取 | 自动提取能力 |
| PromptLayer | Prompt 日志、分析 | 从文档提取 | 文档解析能力 |
| Vellum | 可视化 Prompt 构建 | 从文档提取 | 自动化提取 |
| Notion AI | 文档摘要、问答 | 提取为结构化 Skill | 输出为 Skill 格式 |

## 成功标准

- MVP 发布后 1 周内，团队内部使用
- 从 10 份不同类型的文档中验证提取效果
