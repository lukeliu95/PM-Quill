# Spec 撰写规范

## 格式

- 所有 Spec 文件使用 Markdown
- 文件名小写，用连字符分隔：`my-project-name`
- 每个项目在 `specs/` 下有独立子目录

## 内容规则

- 核心功能不超过 3 个 — 超过就是没想清楚
- "不做什么"至少列 3 条
- 每个功能必须有"成功标准"
- 用户描述要具体到能在路上认出来
- 竞品分析至少包含 2 个竞品

## 语言规则

- 用中文写面向用户的内容
- 技术术语保持英文
- 不用"赋能"、"抓手"、"闭环"这类废话
- 一句话能说清的不写两句

## 文件命名

```
specs/{project-name}/
├── spec.md          # 产品规格
├── feasibility.md   # 可行性分析
├── plan.md          # 执行计划
└── review.md        # 项目回顾（完成后）
```
